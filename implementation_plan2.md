# Comprehensive Debugging Setup Plan

This plan details the steps to enable comprehensive debugging across your full-stack application (frontend, backend, database queries, and VS Code IDE).

## Proposed Changes

### Frontend (Next.js & NextAuth)
- Enable `debug: true` in the NextAuth configuration to see detailed logs of the authentication flow.
- Add debug `console.log()` statements to key pages like `profile/[id]/page.tsx` and API routes to demonstrate route-level debugging.

### Backend (NestJS & Prisma)
- Create a `PrismaService` that instantiates the Prisma Client with query logging enabled (`log: ['query', 'info', 'warn', 'error']`). This will output all SQL queries to the terminal.
- Register `PrismaService` in `AppModule` so it's available throughout your NestJS backend.

### VS Code Debugging Configurations
- Create a `.vscode/launch.json` file at the root of your workspace. This will provide you with easy, one-click debug configurations in VS Code:
  - **"Debug Next.js (Frontend)"**: Attaches a debugger to your Next.js application, enabling you to set breakpoints in your React components and API routes.
  - **"Debug NestJS (Backend)"**: Attaches a debugger to your NestJS server, enabling breakpoints in your controllers and services.

## Verification Plan
1. **VS Code**: You should be able to open the "Run and Debug" panel in VS Code and see the new configurations.
2. **Frontend Logs**: Visiting the application and authenticating will output detailed NextAuth logs to the terminal. Visiting the profile page will output our new console logs.
3. **Backend Logs**: Making database queries through the backend will output the exact SQL queries generated by Prisma to the NestJS console.
